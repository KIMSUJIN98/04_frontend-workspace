8. 속성, 인스턴스
- 속성 : 엔티티에서 관리해야 할 최소단위 정보 항목 => 예를들면 EMP_ID, EMP_NAME, ...	* DB의 테이블
- 인스턴스 : 엔티티의 속성으로 실제로 구현된 값 => 예를들면 선동일, 201, J5				* DB의 값

9. PK(주식별자), FK(외래식별자)
- PK : 엔티티 내 각 인스턴스를 구별하는 기준이 되는 속성								* 테이블 내 각 값을 구별하는 기준이 되는 컬럼
- FK : 관계가 있는 엔티티 간의 연결고리 역할을 하는 속성								* 관계가 있는 테이블 간의 연결고리 역할을 하는 속성(컬럼)

10. 엔티티(테이블)		* 엔티티(테이블)의 특징을 써라?
- 업무의 관심 대상이 되는 사물이여야 함.. => 직원, 급여, 상품
- 두 개 이상의 인스턴스(컬럼의 값)를 소유해야 한다. => 선동일, 노옹철, 하이유		* 하나의 값이 아닌 다중의 값을 소유해야 한다.
- 마땅한 속성을 소유해야 한다 => 컬럼이 있어야 한다.

11. 속성(컬럼)의 명명규칙
- 의미가 분명해야 한다. => EMP_NAME(O), A(X) ??
- 해당 업무에서 사용하는 이름을 부여한다.
- 서술식이나 약어를 사용하지 않는다. EN(X) => EMP_NAME(O)
- 엔티티(테이블)에서 유일하게 식별 가능하도록 지정한다 => 컬럼 중복 X

13. 식별관계 VS 비식별관계
-- 찜 테이블 => ERD 클라우드 TEST ??		* 식별과 비식별은 찜테이블로 설명한 것이 있는데 이를 참조하여 아래 내용 기억해두기!

- 비식별관계 : 외래식별자(FK)가 단순한 속성(컬럼)으로 추가되는 관계

- 식별관계 : 외래식별자(FK)가 주식별자의 일부가 되는 관계
